# Telegram-бот для доступа в закрытую группу

Простая тестовая схема:
- человек выбирает тариф;
- переводит деньги на карту;
- нажимает `Проверить оплату`;
- админу приходит заявка;
- админ вручную подтверждает или отклоняет оплату;
- после подтверждения бот дает кнопку входа в группу;
- когда срок закончился, бот в 10:00 по Киеву удаляет пользователя из группы и снова показывает стартовое меню.

## Что внутри

- `app/main.py` — запуск бота через polling
- `app/handlers.py` — команды, кнопки, заявка админу, подтверждение оплаты, вход в группу
- `app/jobs.py` — ежедневная проверка и удаление просроченных
- `app/db.py` — работа с PostgreSQL
- `app/texts.py` — все тексты и тарифы в одном месте
- `app/keyboards.py` — все кнопки
- `app/config.py` — чтение переменных Railway

## Переменные Railway

```env
BOT_TOKEN=
ADMIN_ID=
GROUP_ID=
CARD_NUMBER=
CARD_HOLDER=
DATABASE_URL=
TZ=Europe/Kyiv
```

### Что значит каждая переменная

- `BOT_TOKEN` — токен Telegram-бота
- `ADMIN_ID` — Telegram ID админа, которому будут приходить заявки на оплату
- `GROUP_ID` — ID закрытой группы
- `CARD_NUMBER` — карта для оплаты, которую бот показывает в тарифе
- `CARD_HOLDER` — имя получателя (необязательно, можно оставить пустым)
- `DATABASE_URL` — PostgreSQL от Railway
- `TZ` — часовой пояс, по умолчанию `Europe/Kyiv`

## Как развернуть на Railway

### 1. Создай проект
- загрузи этот проект в GitHub;
- создай новый проект в Railway;
- подключи репозиторий;
- добавь PostgreSQL.

### 2. Заполни Variables
Вставь переменные из `.env.example`.

### 3. Start Command
Для сервиса поставь такой start command:

```bash
python -m app.main
```

## Что нужно сделать в Telegram

### 1. Бот должен быть админом в группе
У него должны быть права:
- принимать заявки на вступление;
- создавать invite link;
- удалять участников.

### 2. Узнать ADMIN_ID
Напиши боту команду:

```text
/myid
```

### 3. Узнать GROUP_ID
Добавь бота в нужную группу и отправь в группе команду:

```text
/chatid
```

## Как тестировать

### Проверка клиента
1. Нажми `/start`
2. Выбери тариф
3. Нажми `Проверить оплату`
4. У админа должна появиться заявка
5. Нажми `Подтвердить оплату`
6. Клиент должен получить 2 сообщения
7. Нажми `Перейти в группу`
8. Бот должен одобрить заявку в группу
9. Второе сообщение со ссылкой должно удалиться

### Проверка удаления по сроку
Для быстрого теста можно в базе вручную поставить `access_until` на вчерашнюю дату.
На следующем запуске ежедневной задачи бот удалит пользователя из группы и пришлет стартовое меню.

## Где менять суммы потом

Открой файл:

```text
app/texts.py
```

Там есть блок:

```python
PLANS = {
    "1m": {...},
    "3m": {...},
    "12m": {...},
}
```

Меняешь там:
- название тарифа;
- цену;
- количество дней доступа.

## Важно

Сейчас это **тестовый вариант** под ручную оплату на карту.
Автосписаний, платежных callback и веб-сервиса тут нет.
